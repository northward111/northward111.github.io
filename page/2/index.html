JZJModule接入指南

1.将JZJModule库引入工程

源码形式

直接将源码形式的JZJModule库加入工程中，则工程中使用到JZJModule功能的地方加上#import "JZJModule.h"即可

framework形式

通过静态库framework的方式加入工程中，则工程中使用到JZJModule功能的地方加上#import <JZJModule/JZJModule.h>即可

MBA外接显示器后关闭自带显示器

MacBook Air屏幕太小，外接了一个显示器，感觉好多了，顺带配了外接的键鼠。因此，MBA本身就可以安安静静地一边呆着了，本身我是直接把它的盖子合上放到一边的，不过搜索了一番发现盖子合上之后会影响散热，甚至有的说会直接烘烤屏幕，这问题就大了，但是打开盖子之后，也不能直接设置屏幕只投射到外接显示器上，要么把外接显示器作为主显示器，内置的作为扩展屏幕，要么就是镜像屏幕，都不是我要的效果。

网友的力量是强大的，发现两个解决方案：

  1. 拿个磁铁放到电源接口附近，让电脑的感应器误以为已经合上盖子了，屏幕自然就熄了；
  2. 在mac上调整屏幕亮度到最低，会发现内置显示器亮度调到了最低，基本可以看做熄了，同时外置显示器不受影响。

磁铁的方案据说可能对电脑有损伤，所以还是第二个方法比较靠谱，推荐给大家。

About

关于此主页

发现github主页挺适合做个人博客的，但是发现在这上面的文章不能被百度检索到，因此在国内的环境下，写了文章想要分享出去，只放到github上还是挺艰难的。后来发现简书是一个好东西，可以很方便地进行写作与分享，但是文章放到第三方平台上多多少少还是会有风险的，万一哪天简书重心从博客上移走了呢，甚至万一倒闭了呢。。所以各有各的优点，都用一用吧。

关于Hexo

Hexo是一个个人博客发布工具，框架。作者只用把markdown格式的文章写好，丢到指定目录下，就可以用hexo的命令更新网页文件，然后再用hexo的发布命令将网页文件同步到github主页对应的repository下，就是这么轻松。Hexo搭建以及与github主页关联的步骤这里就不赘述了。而现在想要新写一篇文章的步骤可以列为以下步骤：
1.进入terminal，切换目录到hexo的目录下，我的是“/Users/jzj/Documents/hexo_config”
cd /Users/jzj/Documents/hexo_config
2.用hexo新建一个文档
hexo new doc1
3.在hexo目录下的文档子目录下找到刚刚新建的文章，我的是“/Users/jzj/Documents/hexo_config/source/_posts/doc1.md”用markdown编辑器进行编辑
4.编辑完md文档后，在terminal中用hexo生成网页文件
hexo generate
5.用hexo将生成的网页文件同步到github中
hexo deploy
